# MDP WorldComponent 테스트 목록

게임 시작 후 아래 순서대로 테스트. 각 항목에서 Player.log도 확인.

---

## 테스트 1: 기본 동작 확인 (기존 기능 회귀 없음)

1. 월드맵에서 아무 타일 선택 → MapGen AI 대화 열기
2. 채팅창에 입력: `왼쪽에 산 있는 맵 만들어줘`
3. 프리뷰에서 왼쪽에 산이 있는지 확인
4. "맵 생성" 버튼 클릭
5. **확인**: 맵에 왼쪽 산이 정상적으로 생성됨

---

## 테스트 2: MDP 핵심 — "LLM이 안 보낸 것은 유지"

1. 대화 열기
2. 입력: `왼쪽에 산이 있고 나무 많은 맵`
3. 프리뷰 확인 (왼쪽 산 + 식생 많음)
4. 입력: `오른쪽에도 산 추가해줘`
5. **확인**: 왼쪽 산이 유지되면서 오른쪽에도 산이 추가됨
   - Player.log에서 `explicit=` 숫자 확인 (elevation_shapes만 있어야 함)
   - 식생 밀도도 이전 값 유지되어야 함

---

## 테스트 3: MDP 핵심 — "강 방향 유지"

1. 강이 있는 타일 선택 → 대화 열기
2. 입력: `강을 왼쪽으로 흐르게 해줘`
3. 프리뷰 확인 (강 방향 변경됨)
4. 입력: `온천도 추가해줘`
5. **확인**: 강 방향이 왼쪽으로 유지되면서 온천 추가됨
   - LLM이 river 관련 파라미터를 안 보냈어도 기존 강 방향 유지

---

## 테스트 4: 대화 닫기 = 취소

1. 대화 열기
2. 입력: `산 많은 맵`
3. 프리뷰에서 산 확인
4. X 버튼으로 대화 닫기 (맵 생성 안 함)
5. 같은 타일에서 다시 대화 열기
6. **확인**: 이전 "산 많은 맵" 설정이 사라지고, 초기 상태로 돌아감

---

## 테스트 5: Undo 동작

1. 대화 열기
2. 입력: `왼쪽에 산`
3. 입력: `호수도 추가`
4. Undo 버튼 클릭
5. **확인**: 호수가 사라지고 왼쪽 산만 남음
6. Undo 한 번 더
7. **확인**: 초기 상태로 돌아감

---

## 테스트 6: Reset 동작

1. 대화 열기
2. 여러 설정 적용
3. Reset 버튼 클릭
4. **확인**: 모든 설정이 초기 상태로 돌아감, 프리뷰도 원래대로

---

## 테스트 7: 프리셋 로드

1. 이전에 저장한 프리셋이 있으면 로드
2. **확인**: 프리셋이 정상 적용됨 (explicitKeys 비어있으므로 fullApply)

---

## 테스트 8: 연속 수정 (20턴 시뮬레이션)

1. 대화 열기
2. 순서대로 입력:
   - `왼쪽에 산`
   - `오른쪽에도 산`
   - `가운데에 호수`
   - `나무 많이`
   - `동굴 추가`
3. **확인**: 5번째 수정 후 왼쪽 산, 오른쪽 산, 가운데 호수, 높은 식생, 동굴이 모두 유지됨
4. **확인**: Player.log에서 각 Apply마다 기존 값이 유지되는지 확인

---

## 테스트 9: Player.log 확인 포인트

- `[MapGenAI] 파라미터 적용 (tile=XXX, explicit=N)` 로그가 뜨는지
- explicit 숫자가 LLM이 보낸 키 수와 일치하는지
- 기존 값이 유지되는 필드가 로그에 올바르게 표시되는지

---

## 알려진 제한사항

- elevation_shapes는 LLM이 보내면 **전체 교체**. LLM이 기존 shapes를 빠뜨리면 사라짐.
  → 프롬프트에서 "기존 목록 전체 복사 + 새 항목 추가" 규칙으로 완화
  → elevation_shapes를 안 보내면 기존 유지 (explicitKeys에 없음)
- 세이브/로드 테스트는 이번에 안 함 (WorldComponent ExposeData는 구현했지만 게임 저장/로드 주기 테스트 필요)
